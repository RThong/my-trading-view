# My Trading View 用户手册

这是一款**只在你自己电脑上运行**的金融市场看板，帮你每天用一眼时间看清美股市场处于"恐慌、平静还是过热"。

## 它能为你做什么

- **每天自动给你抓取关键市场数据**：不用打开十几个网页一个一个看，开机就能看到最新的全市场状态
- **一张图看完恐慌指数、利率、股指、黄金、比特币**：所有你关心的资产挤在一个面板里，互相对照
- **完整保留十几年历史**：随手切到 1Y / All，能直接对比当前和 2008、2020 那种危机时刻
- **零订阅费用、数据不出你电脑**：不用付 Bloomberg、不用 TradingView Pro，所有原始数据存在你自己的硬盘
- **专门给"看趋势"优化**：周末/假期自动隐藏空白，把图表压紧，跟专业终端的视觉一致

## 功能详细介绍

### 波动率面板（Volatility）

**是什么**：把美股市场的"恐慌指数家族"全部塞在一张图里——VIX、9 天 / 1 月 / 3 月 / 半年 / 1 年的不同期限恐慌指数、VIX 的波动率（VVIX）、SKEW 黑天鹅指数、以及 VIX 期货 VX1。

**能帮你做什么**：判断市场情绪。
- 9 天 VIX 突然冲到 30 天 VIX 之上，说明短期出事了
- SKEW 飙升说明有人在大量买"黑天鹅保险"，市场暗流涌动
- VX1 比 VIX 现货高很多（contango），说明远期更担心；反过来则是当下更恐慌

**怎么找到它**：顶部 Tab 第一个，叫 `Volatility`。

### 宏观利率面板（Macro / Rates）

**是什么**：3 种美债收益率（10 年、2 年、3 个月）和美元指数，全部画在一张图上。

**能帮你做什么**：
- 看美联储紧缩还是放水：短端利率升说明紧、降说明放
- 看"收益率倒挂"：2 年高过 10 年通常预示衰退
- 美元指数走强通常压制风险资产

### 股指面板（Indices）

**是什么**：标普 500（大盘）、QQQ（科技）、IWM（小盘）三个分行展示。

**能帮你做什么**：一眼看出大盘 / 科技 / 小盘谁强谁弱。一般来说牛市后半段小盘开始领涨，熊市开始时大盘还撑着但小盘先崩。

### 其他资产面板（Other Assets）

**是什么**：黄金（GLD）、长债（TLT）、比特币（BTC）三个独立分行。

**能帮你做什么**：看"避险资产"。市场出事时黄金涨、长债涨；BTC 现在越来越像高 beta 的科技股，跟纳指联动。

### 期权偏度面板（Options 25Δ）

**是什么**：每天追踪 SPY 的 25-delta 看涨/看跌期权隐含波动率，以及它们的差值（偏度）。

**能帮你做什么**：判断市场对"上涨"和"下跌"的恐惧不对称程度。
- SPY 偏度（put IV 减 call IV）大 = 大家在抢着买跌保护

> ⚠️ 这个面板从你**第一次运行**那天开始积累数据，没有历史。一个月后才有 20 个点，三个月后才有意义。

### 时间周期切换

顶部右上角的 6 个按钮：**1D / 1W / 1M / 1Q / 1Y / All**。

- **1D**：原始日线，每根线代表一个交易日
- **1W / 1M / 1Q**：把多天聚合成一根（取该周期最后一天的收盘价）
- **All**：拉到最早（2007 年 1 月）

切换是全局的——切到 `1Y`，所有 5 个 Tab 的图都会一起变。

### 数据状态灯

顶栏左上角有个小圆点：

- 🟢 **绿色**：最近一次后台数据拉取一切正常
- 🟡 **黄色**：部分数据源失败（比如有 2 个 ticker 没拉到）
- 🔴 **红色**：完全失败（可能是网络问题或者 FRED key 失效）
- ⚪ **灰色**：还没运行过

鼠标悬停可以看具体哪个数据源出问题。

## 使用步骤 / 操作指南

### 如何第一次启动看板

1. 打开 terminal，进入项目目录
2. 输入 `bun install` 等它装完依赖
3. 去 [FRED 网站](https://fred.stlouisfed.org/docs/api/api_key.html) 免费申请一个 API key（30 秒）
4. 复制 `.env.example` 为 `.env`，把 key 填进去
5. 输入 `bun run db:migrate` 建数据库
6. 输入 `bun run job:daily` 第一次拉数据（含历史，要等几十秒到一两分钟）
7. 输入 `bun run dev`——一行命令同时跑前端和后端
8. 浏览器打开 `http://localhost:5173`

💡 小提示：第 7 步的 `bun run dev` 会在一个 terminal 里同时启动前端和后端，按 Ctrl-C 一次性关闭两个。

### 如何让看板每天自动更新

1. 在 terminal 里输入 `./scripts/install-launchd.sh`
2. 系统会自动配置好"每天早上 8 点（你电脑的本地时间）自动拉一次数据"
3. 完成。每天打开电脑就有最新数据，不用手动跑命令

💡 小提示：如果你电脑那个时间是睡眠状态，macOS 会在你唤醒电脑后自动补跑一次。

### 如何切换看不同的指标

1. 顶部 Tab 栏点 `Volatility` / `Macro / Rates` / `Indices` / `Other Assets` / `Options (25Δ)`
2. 切 Tab 时图表瞬间替换，时间范围保持不变
3. 想看更长历史就点右上角 `All`

### 如何在指标之间对照

每个面板里多条线**默认同图**，颜色不同。鼠标在图上移动会显示十字光标和每条线在当前日期的具体数值。

💡 小提示：左轴和右轴可能不同——比如波动率面板里，VIX 系列在左轴（10-80 范围），VVIX 和 SKEW 在右轴（80-180 范围）。鼠标停下来读数值才是准的。

### 如何手动重新抓一次数据

如果你想立刻拿到最新数据（不等明天），直接：

```
bun run job:daily
```

跑完会显示 `Daily job complete.`，刷新浏览器就能看到。

## 常见问题 FAQ

**Q：这个工具是免费的吗？**
A：是。所有数据源（Yahoo Finance、CBOE 公开数据、FRED）都免费。你不需要付任何订阅费。

**Q：我的数据存在哪里？会被人看到吗？**
A：所有数据都存在你电脑里的一个文件 (`data/mtv.db`)。这个工具默认不上传任何东西到外面，也不需要登录账号。

**Q：FRED 那个 API key 安全吗？**
A：放在 `.env` 文件里，这个文件已经在 git 忽略列表中，不会被提交到 GitHub。即使万一泄露，FRED 的 key 也只能用来读公开经济数据，没法做任何"敏感"的事，吊销重发也是 30 秒。

**Q：周末/假期看板会显示什么？**
A：周末美股不交易，所以看板显示的是最近一个交易日的数据（通常是周五）。X 轴上不会出现周六周日的空白，这跟专业终端的做法一致。

**Q：我电脑关机/休眠了一晚，第二天数据会不会缺一天？**
A：**大部分数据会自动补齐**，但 SPY 期权例外。详见下一条。

**Q：连续关机好几天会发生什么？**
A：分两种情况：

- **股指 / 利率 / 波动率指数 / VIX 期货等**：✅ 全部能补。系统下次开机时会拉取"上次成功日 ~ 今天"之间所有缺失的工作日，无缝填补。
- **SPY 25Δ 期权**：❌ **不能补**。期权数据源（moomoo OpenD）只提供"当前快照"，没有任何接口能查"3 天前的期权链"。所以你关机那几天的期权数据点是永久丢失的——曲线上会显示成断点。

💡 小提示：如果你长期出差关电脑，可以在出差前手动跑一次 `bun run job:daily` 拿到当天数据。但是回来后中间那几天的期权数据无论如何补不回。

**Q：可以在手机上看吗？**
A：目前不行——这个工具是给本地电脑（macOS）设计的。如果你坚持要手机看，需要把电脑暴露到公网（不推荐，给个人项目不值得）。

**Q：能加我自己关心的指标吗（比如美元/日元、纳指、中证 500）？**
A：能。Yahoo Finance 上有的标的几乎都能加——告诉懂代码的人，改一处配置文件就行。

**Q：为什么 SPY 期权 Tab 几乎是空的？**
A：期权数据源只给"当前快照"，没有历史。这个 Tab 从你第一次启动那天开始累积，每天加一个点。一个月后才会看出短期形态，建议先放着养着。

**Q：图表为什么看起来 Lag/迟钝？**
A：通常不会，但如果你的 `All` 视图同时显示几万个数据点 + 6 条线，浏览器有点压力。换到 `1Y` 或 `1Q` 就立刻流畅。

**Q：状态灯一直红的，怎么办？**
A：鼠标停上去看具体哪个数据源失败。常见原因：
- `macro: FRED_API_KEY is required` → 你忘了配 `.env` 里的 FRED key
- `quotes: Failed to parse JSON` → Yahoo Finance 临时抽风，等一天或重跑 `bun run job:daily` 试试
- 全部失败 → 你电脑没联网

**Q：可以多久不维护？**
A：理论上可以一直放着自动跑。但 Yahoo 这种爬虫数据源偶尔会变接口（比如几个月前 v2 停摆了），到时候可能需要更新一下 `yahoo-finance2` 库版本。

## 应用场景举例

### 场景 1：开盘前 5 分钟的状态扫描

小李是一位**业余美股投资者**，每天早上 8 点（日本时间）起床想看看"昨夜美股发生了什么"，但又懒得打开 5 个网站。

以前他要分别查 VIX、查 10 年期美债、看一眼 BTC、刷一遍 SPX 走势——光是浏览器开标签就要 3 分钟。

用了 `My Trading View` 之后，他打开浏览器一个本地链接，**所有 5 个面板一屏全在**，VIX 突破 20 立刻看到，10Y 利率有没有跳一目了然。**整个流程从 3 分钟压缩到 30 秒**。

### 场景 2：市场大跌时的"历史对比"

老王是一位**有 10 年经验的交易员**，3 月某天美股突然跳水 4%，他想知道"这次跟 2020 年 3 月、2008 年 10 月相比是大事还是小事"。

如果用免费的 Yahoo Finance 网页，最远只能看 5 年——比不上 2008。

用 `My Trading View` 切到 `All`，**VIX 历史最高的两根尖刺**（2008-11-20 的 80.86 和 2020-03-16 的 82.69）**一目了然**——他今天的 VIX 才到 25，跟这两根比还差得远。**他放心了，没有清仓**。

### 场景 3：每周末研究期限结构

阿强是一位**期权策略爱好者**，最近想琢磨"contango / backwardation"现象——VIX 期货比现货贵多少。

每周末他打开看板的 `Volatility` Tab，**VIX 现货线和 VX1 期货线同色系并排**：
- 大部分时间 VX1 略高于 VIX → 正常 contango
- 偶尔 VX1 反而低于 VIX → 那是市场极度恐慌（backwardation）的信号

他做了个表，**用历史 18 个月数据发现：每次 backwardation 出现后 5 个交易日内 SPX 都反弹**。基于这个发现，他做出了一笔风险/收益不错的交易。
